# Loom CLI L3 Feature Tickets Specification

## Overview

This document specifies the feature ticket document structure generated by derive-l3.

**Traceability:** Derived from [acceptance-criteria.md](../l1/acceptance-criteria.md) and [tech-specs.md](../l2/tech-specs.md).

---

## Feature Ticket Structure

### FDT-001: FeatureTicket Entity

**File:** `cmd/derive_l3.go`

```go
type FeatureTicket struct {
    ID                     string   `json:"id"`
    Title                  string   `json:"title"`
    Status                 string   `json:"status"`
    BusinessGoal           string   `json:"business_goal"`
    UserStory              string   `json:"user_story"`
    AcceptanceCriteriaRefs []string `json:"acceptance_criteria_refs"`
    NFR                    []string `json:"nfr"`
    Dependencies           []string `json:"dependencies"`
    ImpactAreas            []string `json:"impact_areas"`
    OutOfScope             []string `json:"out_of_scope"`
    Priority               string   `json:"priority"`
    EstimatedComplexity    string   `json:"estimated_complexity"`
}
```

**ID Pattern:** `FDT-{NNN}`

Where:
- `{NNN}` - Sequential ticket number (001, 002, ...)

**Example:** `FDT-001`

**Related DEC:** DEC-022 (L3 document ID prefixes)

---

## Field Definitions

### Status Values

| Status | Description |
|--------|-------------|
| `draft` | Initial state, under review |
| `ready` | Ready for implementation |
| `in_progress` | Currently being implemented |
| `done` | Implementation complete |

### Priority Values

| Priority | Description |
|----------|-------------|
| `critical` | Must be in MVP |
| `high` | Important for launch |
| `medium` | Nice to have |
| `low` | Future consideration |

### Complexity Values

| Complexity | Description |
|------------|-------------|
| `trivial` | < 1 day |
| `simple` | 1-2 days |
| `medium` | 3-5 days |
| `complex` | 1-2 weeks |
| `very_complex` | > 2 weeks |

## Output Format

### OUT-001: feature-tickets.md

**File:** `{output-dir}/feature-tickets.md`

**Structure:**
```markdown
---
title: Feature Definition Tickets
generated: 2024-01-15T10:30:00Z
status: draft
level: L3
---

# Feature Definition Tickets

---

## FDT-001: Order Creation

**Status:** draft | **Priority:** high | **Complexity:** medium

### Business Goal
Enable customers to place orders through the platform.

### User Story
As a customer, I want to create orders so that I can purchase products.

### Acceptance Criteria References
- AC-ORD-001
- AC-ORD-002

### Non-Functional Requirements
- Response time < 500ms
- Support 1000 concurrent orders

### Dependencies
- Customer authentication
- Inventory service

### Impact Areas
- Order database
- Notification service

### Out of Scope
- Payment processing (separate ticket)
- Shipping calculation

---
```

## Generation Process

### SEQ-FDT-001: Feature Ticket Generation

**Input:**
- `acceptance-criteria.md` (L1)
- `tech-specs.md` (L2)

**Prompt:** Uses `prompts.DeriveFeatureTickets`

**Process:**
1. Read AC and tech specs
2. Generate feature tickets via Claude
3. Parse ticket structure
4. Format as markdown

**Output:** `feature-tickets.md`
